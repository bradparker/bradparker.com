Front end web development

I've done a few versions of front end development, from Wordpress themes to universal React applications.

Markup

I've been writing HTML since 1996 when I put together a few dinky Geocities sites. I always have the generated HTML in mind when using tools that abstract over it, this includes everything from Markdown to React. I feel like that can be helpful, there's a lot that the browser can help you with if you give it the right hints. From reasonably simple stuff like viewport meta through to making sure that custom form controls remain usable for keyboard and screen reader users.

Interaction

JavaScript is the language I've used most. For a long time it served as the language I thought about problems in. I've used it to write reasonably large single page applications. I wrote some documentation for an early version of one of these single page apps which might be illustrative.

I first started to use React extensively in early 2015. We chose to replace Outfit's Angular 1 prototype (which I had also written) with a Rails app decorated with React components. We were even able to achieve progressive enhancement by using a pretty nifty gem to render components on the server side as part of Rail's usual view rendering process. I thought that it was all pretty amazing.

I got write a lot of code using React at everydayhero. All new user interfaces were written in React, we had our own component libraries and tooling to make building universal apps and static sites a bit easier.

Presentation

Modern CSS is amazing. The tools we have now for managing layout, for example, work wonderfully and the consistency across browsers is better than ever. That being said name spacing is still hard. I've worked with a few different convention-based solutions like Block Element Modifier and Atomic CSS. I've also worked with tools that try to automate scoping like CSS Modules and CSS in JS.

Back end web development

At both NetEngine and everydayhero I got to kick off brand new Ruby on Rails apps and maintain well-used older ones. A Rails application that has stuck close to convention can be a very pleasant thing to work on.

I've deployed a few web services in Node.js, it went well. It's good to have an ecosystem of full of packages which all compose pretty nicely.

Despite not using it professionally, over the past few years I've written a lot of Haskell. At Stacktrace we maintained regular Friday afternoon learning sessions, so far we've worked through Brent Yorgey's CIS 194 lecture series and QFPL's three day functional programming workshop. Recently I implemented Thinkster's RealWorld API specification in Haskell, as a fun learning exercise.

Design

I got my start in web development through being hired as a designer for a software company. I have designed user interfaces, visual identities and promotional material. I really love great typography and good, sound structure.
