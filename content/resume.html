Resume

I've been working as a web developer and designer for the past seven years. In that time I've been lucky enough to work up and down the stack with some really amazing people. Being able to learn from and with both colleagues and clients is one of the many reasons I love doing what I do.

Skills

I'm curious and try to keep an open mind. Changing careers from music education, to design, to software engineering has made me comfortable with being a beginner.

Below I'll summarise my experience with the technologies I've used most or am most passionate about.

HTML and CSS (including generators like SASS and CSS Next)

Like a lot of people I had a Geocities site when I was around ten years old (I wrote all tags in SHOUTY CASE, which I miss to some extent). Later, when working on front ends professionally, I had to learn about taming the complexity of CSS through preprocessors like SASS and patterns like BEM and Atomic CSS.

JavaScript (including Node.js)

JavaScript is the language I've used most. For a long time it served as the language I thought about problems in. I've been writing it professionally since I started out in web development.

I've used it to write developer tooling, reasonably large single page applications and back end web services. I wrote some documentation for an early version of one of these single page apps which might be illustrative.

React

I first started to use React extensively in early 2015. We chose to replace Outfit's Angular 1 prototype (which I had also written) with a Rails app decorated with React components. We were even able to achieve progressive enhancement by using a pretty nifty gem to render components on the server side as part of Rail's usual view rendering process. I thought that it was all pretty amazing.

I got write a lot of code using React at everydayhero. All new user interfaces were written in React, we had our own component libraries and tooling to make building universal apps and static sites a bit easier.

Ruby (including Rails)

At both NetEngine and everydayhero I got to both kick off brand new Rails apps and maintain well-used older ones.

Haskell

Despite not using it professionally, over the past few years I've written a lot of Haskell. At Stacktrace we've maintained regular Friday afternoon learning sessions, so far we've worked through Brent Yorgey's CIS 194 lecture series and QFPL's three day functional programming workshop. Recently I implemented Thinkster's RealWorld API specification in Haskell, as a fun learning exercise.
